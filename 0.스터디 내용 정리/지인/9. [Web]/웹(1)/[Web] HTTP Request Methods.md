# HTTP Request Methods

HTTP Request Methods는 클라이언트(보통 브라우저)가 서버에 리소스(정보)를 요청하거나 조작할 때 사용하는 표준 방식입니다.

각 메서드는 특정한 동작을 의미하며, 웹 애플리케이션의 다양한 기능 구현에 쓰입니다.

HTTP 통신 과정 역시 이 메서드들을 기반으로 이루어집니다.

<br/>

## HTTP Request Methods 종류

### 주요 요청 메서드

- **GET**
  리소스를 조회할 때 사용합니다.
  - 서버에 전달할 데이터(예: 검색어)는 쿼리스트링을 통해 URI에 포함하여 보냅니다.
  - 요청 본문(body)은 일반적으로 사용하지 않으며, 데이터를 캐싱할 수 있습니다.
  - 예시: **`/members/100?username=kim`**
  - 정적 페이지, 사용자 정보, 이미지·파일, 검색 결과 등 조회 중심 요청에 적합합니다.
- **POST**
  서버에 데이터를 전달하여 리소스를 생성하거나, 데이터 처리를 요청할 때 사용합니다.
  - 요청 데이터는 반드시 요청 본문(body)에 담아 전송합니다.
  - 폼 데이터(**`application/x-www-form-urlencoded`**), 파일 업로드(**`multipart/form-data`**), JSON 등 다양한 포맷 지원.
  - 신규 정보 생성, 양식 데이터 제출, 복잡한 검색 등 서버 상태를 변화시키는 작업에 활용됩니다.
- **PUT**
  지정한 리소스를 “전체적으로 대체”합니다(전체 업데이트).
  - 리소스가 존재하면 덮어쓰고, 없으면 새로 만듭니다.
  - 요청 본문에 전체 데이터를 담아야 하며, 일부만 수정하는 데는 부적합합니다.
- **PATCH**
  리소스의 “일부 속성”만 선택적으로 수정할 때 사용합니다.
  - 요청 본문에 수정할 필드만 포함합니다.
  - 데이터 전체 대체가 아닌, 부분 업데이트에 적합합니다.
- **DELETE**
  특정 리소스를 삭제할 때 사용합니다.
  - 서버는 해당 리소스를 삭제 후, 상태 코드(주로 200 또는 204 No Content)로 응답합니다.

### 기타 메서드

- **HEAD**
  GET과 거의 같지만 본문(body) 없이 헤더만 반환합니다.
  - 리소스 존재 유무, 응답 헤더 정보만 필요할 때 유용합니다.
- **OPTIONS**
  리소스가 지원하는 메서드 및 옵션(주로 CORS 등)을 미리 확인할 때 사용됩니다.
- **TRACE**
  클라이언트-서버 경로상 패킷이 변조되지 않았는지 확인하고자 루프백(Loopback) 테스트를 수행합니다.
- **CONNECT**
  프록시 등에서 SSL 터널(HTTPS 연결 등) 생성 시 사용합니다.

<br/>

## **HTTP 통신 과정**

1. 클라이언트가 서버에 HTTP 요청(Request)을 보냄
   - 요청에는 URI(주소), 메서드(GET, POST 등), 헤더, 바디 등이 포함됨
2. 서버는 요청을 해석한 뒤 적절한 작업을 처리함
   - 데이터베이스 조회, 파일 처리 등
3. 처리 결과를 HTTP 응답(Response)으로 클라이언트에 반환함
   - 응답에는 상태 코드(예: 200 OK, 404 Not Found), 헤더, 바디 등이 포함됨
4. 통신이 종료되며, 이후 필요한 경우 추가 요청이 수행됨

예를 들어, GET 메서드는 쿼리 문자열로 조회 요청을 보내고, 서버가 해당 데이터를 찾아 응답합니다.

POST는 폼 데이터를 바디에 담아 서버로 전송하고, 서버는 데이터베이스에 결과를 저장해 응답합니다.

PUT은 기존 리소스를 완전히 대체하며, PATCH는 데이터 일부만 수정합니다.

DELETE는 지정한 리소스를 서버에서 제거합니다.
